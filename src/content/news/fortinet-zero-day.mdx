## ðŸ§© Technical Details

The threat actor claims the exploit enables remote code execution without authentication, providing full control over the targeted FortiGate devices.
This includes the ability to extract sensitive configuration files and information such as credentials and security policies.
The exploit's advertisement surfaced as Fortinet released an advisory about attackers maintaining persistent access to devices by exploiting known vulnerabilities, specifically CVE-2022-42475, CVE-2023-27997, and CVE-2024-21762.
Attackers have been creating symbolic links between user and root filesystems to evade detection and maintain access even after patches are applied.

## ðŸ§± Mitigation Guidance

Fortinet recommends the following actions to mitigate risks associated with the exploitation of known vulnerabilities:

- Update FortiOS to versions 7.6.2, 7.4.7, 7.2.11, 7.0.17, or 6.4.16, which include fixes for the symbolic link issue.
- Apply the latest AV/IPS signatures provided by Fortinet.
- Audit SSL-VPN configurations and monitor for unauthorized symbolic links or unusual access patterns.
- Implement strict access controls and monitor administrative account activities.

As the alleged zero-day exploit has not been officially confirmed or assigned a CVE ID, organizations should remain vigilant and monitor Fortinet's advisories for updates.
